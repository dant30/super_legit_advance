Project Structure
Project Root: super_legit_advance
Apps:

Loans: Manages loan-related functionalities.
Customers: Manages customer data and interactions.
Reports: Generates various reports related to loans and payments.
Notifications: Handles notifications and reminders for payments.
Core: Contains shared logic, utilities, and management commands.
Key Components
Models:

Repayment: Tracks loan repayments with various statuses (e.g., Pending, Completed, Overdue).
Penalty: Manages penalties for late payments.
MpesaPayment: Tracks M-Pesa payment requests and results.
Views:

Repayment API Views: Includes views for creating, updating, canceling, and waiving repayments.
M-Pesa API Views: Handles M-Pesa payment requests and callbacks.
Serializers:

RepaymentSerializer: Serializes repayment data for API responses.
MpesaPaymentSerializer: Serializes M-Pesa payment data.
Management Commands:

send_reminders.py: Sends reminders for upcoming payments and overdue items.
Utilities:

RepaymentCalculator: Contains logic for calculating repayment amounts, interest, and penalties.
Functionality
Repayment Management: Users can create, update, cancel, and waive repayments through the API.
M-Pesa Integration: The system can initiate payments via M-Pesa and handle callbacks for payment status updates.
Reporting: The application can generate reports summarizing loans, payments, and customer interactions.
Notifications: The system sends reminders for upcoming payments and overdue items.
